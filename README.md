# EM

Кроссплатформенный C++ проект с поддержкой сборки на **Windows** и **Linux/WSL** через **CMake**.  
Проект использует **CMake Presets** для конфигураций **Debug** и **Release**, ведёт логирование через **glog** и работает с локально поставляемыми архивами зависимостей, которые распаковываются при первом запуске подготовки окружения.

## Возможности

- Сборка через **CMake**
- Готовые пресеты для:
  - Windows x64 Debug / Release
  - Windows x86 Debug / Release
  - Linux Debug / Release
- Кроссплатформенная разработка:
  - Windows
  - Linux
  - WSL / удалённая Linux-среда
- Логирование через **glog**
- Локальные зависимости внутри репозитория
- Автоматическая распаковка архивов зависимостей при первичной подготовке проекта
- Сборка через **Ninja**

## Архитектура проекта

Проект разделён на несколько уровней ответственности, чтобы структура данных и логика ввода-вывода были независимы друг от друга.

### `ListNode`
`ListNode` — базовая структура узла двусвязного списка.

Содержит:
- `prev` — указатель на предыдущий узел
- `next` — указатель на следующий узел
- `rand` — указатель на произвольный узел списка или `nullptr`
- `data` — строковые данные узла

Структура `ListNode` является общей для всех остальных модулей и не изменяется.

---

### `List`
`List` — контейнерный модуль, который управляет узлами `ListNode`.

Он отвечает за:
- хранение начала и конца списка
- подсчёт количества элементов
- вставку и удаление узлов
- поддержку связей `prev` и `next`
- освобождение памяти

`List` не знает ничего о формате входного файла и не выполняет сериализацию.

---

### `ListSerializer`
`ListSerializer` — модуль ввода-вывода.

Он отвечает за:
- чтение текстового файла `inlet.in`
- построение списка `List` по входным данным
- сериализацию списка в бинарный файл
- десериализацию бинарного файла обратно в `List`

Для восстановления поля `rand` используется двухэтапный алгоритм:
1. сначала создаются все узлы списка
2. затем отдельным проходом восстанавливаются произвольные ссылки `rand`

---

### `main.cpp`
`main.cpp` — точка входа в программу.

Отвечает за:
- создание объекта `List`
- вызов методов `ListSerializer`
- запуск сценария:
  - чтение `inlet.in`
  - построение списка
  - сериализация в `outlet.out`
